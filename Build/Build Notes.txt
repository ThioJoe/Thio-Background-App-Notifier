Command:
	wix build ThioBackgroundAppNotifier_Installer.wxs -arch x64 -loc InstallerText.wxl -ext WixToolset.UI.wixext -ext WixToolset.Util.wixext -o "Thio-Background-App-Notifier_Installer.msi" -d ProductVersion="1.x.x.0"
	
	(-loc InstallerText.wxl overrides the built-in WixUI wording on the install-scope page.
	 If you drop that flag the installer still builds, but reverts to WiX's default obtuse text.)



To get Wix:
	- You can get the wix toolset from this repo's releases: https://github.com/wixtoolset/wix
	- You can install it, or just download the "artifacts.zip" and extract the nupkg files and it will run portably

========== To use Wix portably without needing to install ===========
	
	1. Download artifacts.zip from the github repo linked above. Get the latest V6 one.

	2. Look for the "main" nupkg based on the version name, for example for v6.0.2 it's "wix.6.0.2.nupkg"

	3. Extract with 7-zip or something. The actual program files we need (such as wix.exe) are at "tools\net6.0\any"
		- Copy the entire "any" folder somewhere more convenient and rename it to whatever you want.
		- Then instead of running the command like "wix build whatever" you could do:   "C:\whatever path\wix.exe" build whatever
		
	4. For the extensions, back in artifacts.zip look for the ".wixext." nupkg files and extract the ones for the needed extensions
		- After extracting, find the dll for the extension. For V6 they are probably in the "wixext6" folder. There should only be one dll per extension.
			- You can put the dll next to the wix.exe you got before.
		- To use the extensions, for each "-ext" parameter in the build command, you need to use the full path to the extension dll even if they are right next to wix.exe

========== Notes for installed version of wix ==============

Check wix version
	wix --version

Install specific extension versions via:
	wix extension add -g WixToolset.UI.wixext/6.0.0
	wix extension add -g WixToolset.Util.wixext/6.0.0


Install extensions in general via:
	wix extension add -g WixToolset.UI.wixext
	wix extension add -g WixToolset.Util.wixext

Per-user vs per-machine install (WiX 6, Package Scope="perUserOrMachine")
	The installer is now dual-purpose. It defaults to a per-user install so no admin rights are needed.

	- Double-click / run the MSI normally (not elevated):
		Installs "just for me" -> %APPDATA%\ThioJoe\Thio's Background App Notifier
		(AppDataFolder is Roaming %APPDATA%.) The scope dialog only shows the "just for me" text.

	- Run the MSI elevated (right-click -> Run as administrator, or an elevated prompt):
		The scope dialog shows the "all users / just me" radio choice (defaults to all users).
		"All users" installs to C:\Program Files\ThioJoe\Thio's Background App Notifier (as before).

	Silent installs:
		Per-user:     msiexec /i "Thio...msi" /qn                (or MSIINSTALLPERUSER=1 ALLUSERS=2)
		Per-machine:  msiexec /i "Thio...msi" /qn ALLUSERS=1     (must run from an elevated prompt)

	Notes:
	- The app always stores its detection log in %APPDATA%\ThioJoe\Thio's Background App Notifier
	  (DetectionStore.StoreDirectory), regardless of whether the exe was installed per-user or
	  per-machine. For a per-user install that folder is also where the exe lives; for an all-users
	  (Program Files) install the exe is elsewhere but the log still goes to AppData so no admin
	  rights are needed to write it.
	- ICE validation is not run by "wix build" (only by MSBuild), so the single-component layout is
	  kept as-is. If you ever switch to MSBuild and want to silence per-user ICEs (ICE38/ICE43/ICE57),
	  set <SuppressValidation>true</SuppressValidation> or add an HKCU registry KeyPath to the component.
